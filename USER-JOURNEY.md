# BriStack 用户旅程

**产品版本：** MVP
**文档日期：** 2026-02-24
**适用对象：** 内容创作者、订阅者、AI Agent 接入方

---

## 概览

BriStack 是一个为 AI 时代设计的内容分发平台。创作者在这里发布结构化内容，读者通过邮件、网页、AI 助手或 API 多种方式获取。

平台同时服务三类订阅者：
- **人类订阅者**：通过邮件或网页直接阅读
- **AI Agent 订阅者**：通过 API Key 定期拉取结构化内容
- **MCP 订阅者**：将 Space 接入 Claude Desktop 等 AI 客户端，实时检索

---

## 一、创作者旅程

### 1. 注册与初始配置

创作者通过管理员密码登录平台。首次登录时，系统自动创建创作者账户。

登录后，前往 **设置页面** 完成初始配置：

```
落地页设置（显示来自落地页的订阅者数）
├── Space 名称（显示在落地页和邮件中）
├── 个人 Slug（决定落地页 URL，如 /your-slug）
├── 个人简介（显示在落地页）
└── 头像 URL

我的外部网站（每个网站独立展示，显示来自该网站的订阅者数）
├── 添加外部网站（填写网站名称和 URL）
├── 获取嵌入代码（iframe 代码，复制粘贴至网站）
└── 嵌入预览（实时预览订阅表单效果）

邮件服务
├── 选择发件方式：SMTP 或 Resend
├── 填写服务商配置（可测试连接验证是否正确）
├── 发件人名称和邮箱
└── 测试连接（点击按钮即时验证是否可用）

AI 功能
└── Anthropic API Key（用于内容穿透率评分）
```

**订阅来源追踪**：每个渠道（落地页、外部网站A、外部网站B）的订阅者数量在设置页单独显示，帮助创作者了解各渠道获客效果。

### 2. 创建内容

进入 **内容页面** → **新建内容**，进入结构化编辑器：

```
编辑界面包含
├── 标题
├── 副标题（可选）
├── 封面图 URL（可选）
├── 主体内容（富文本编辑器，支持 Markdown 格式）
│
├── AI 时代结构化字段
│   ├── AI 摘要（自动生成，可手动修改）
│   ├── 核心论点（列表形式，最多 5 条）
│   ├── 主题标签
│   └── 作者注释（手写，AI 无法覆盖）
│
├── 元信息
│   ├── 预计阅读时长
│   └── 人类写作声明
│
├── 内容渠道（控制内容在哪些渠道可见）
│   ├── ☑ 公开落地页
│   ├── ☑ 邮件发送
│   └── ☑ 外部网站（每个已添加的网站独立勾选）
│
└── 内容穿透率评分（发布时自动计算）
```

**内容渠道控制**是核心分发管理功能：创作者可为每篇内容单独指定面向哪些渠道开放。例如：
- 某篇内部分析仅发送邮件，不展示在公开落地页
- 某篇适合特定网站受众的内容，仅对该网站的订阅者可见
- 默认（全部勾选）= 对所有渠道开放

**内容穿透率**是平台的核心创新指标：发布后，系统将内容交给 AI 压缩为摘要，计算创作者填写的「核心论点」有多少被摘要保留。目标穿透率 ≥ 70%。

**浏览模式**：编辑完成后可切换至「浏览模式」，预览内容最终呈现效果。

### 3. 内容管理

**内容列表**提供两层筛选，帮助创作者管理大量内容：

```
状态筛选（横向标签）
├── 全部
├── 已发布
└── 草稿

渠道筛选（横向标签，动态生成）
├── 全部渠道
├── 落地页（仅显示在落地页开放的内容）
├── 邮件（仅显示可通过邮件发送的内容）
└── 外部网站A / 外部网站B（按各网站筛选）
```

每条内容支持：
- 点击进入编辑
- 悬停显示操作按钮（撤回为草稿、删除）
- 已发布内容显示穿透率分数和发布时间

### 4. 发布与分发

点击「发布」后，内容根据渠道设置，自动分发至对应渠道：

| 渠道 | 路径 | 说明 |
|------|------|------|
| 落地页内容列表 | `/{slug}` | 需勾选「公开落地页」渠道 |
| JSON Feed API | `/api/v1/feed` | 等同落地页渠道，AI 友好 |
| MCP Server | `/api/mcp` | MCP 订阅者可检索 |
| 邮件发送 | 手动触发 | 需勾选「邮件发送」渠道 |

### 5. 邮件发送

发布后，前往 **发送页面** 将内容推送给邮件订阅者：

```
发送配置
├── 预览发送（向自己发一封测试邮件）
├── 筛选条件
│   ├── 按订阅者等级（Level 1-4 或全部）
│   └── 按标签筛选
└── 正式发送
    ├── 批量发送，每批 10 人，间隔 1 秒（避免触发限流）
    └── 发送结果：成功 N 封 / 失败 N 封
```

邮件内容自动包含：
- 创作者署名头部
- 发布日期和预计阅读时长
- 主题标签
- AI 摘要区块（结构化呈现）
- 完整正文
- 作者注释（如有）
- 退订链接（合规）
- 追踪像素（统计真实打开）

### 6. 订阅者触达序列

**欢迎邮件**（订阅即发送）：
- 可在设置中自定义主题和正文
- 支持变量：`{name}`（订阅者姓名）、`{creatorName}`（创作者名称）
- 包含 MCP 接入配置示例

**问候邮件**（可配置延迟，默认 3 天后）：
- 可启用/禁用
- 完全自定义内容

### 7. 分析与洞察

**双轨分析仪表盘** 严格区分人类流量与 AI/Bot 流量：

```
人类影响力指标
├── 真实人类订阅者数（按来源：落地页 / 各外部网站）
├── 订阅者等级分布（Level 1-4）
└── 标签分布

AI 生态指标
├── Agent 订阅者数
├── MCP 订阅者数
└── 各渠道查询量趋势
```

**Bot 识别机制**：
- UA 特征识别（GPTBot、ClaudeBot、PerplexityBot 等已知爬虫）
- 行为特征检测（访问速度、访问路径）
- 结果标记：`human` / `known_bot` / `suspected_bot`

**订阅者等级系统**（自动升级）：

| 等级 | 含义 | 条件 |
|------|------|------|
| Level 1 | 已订阅 | 完成订阅 |
| Level 2 | 验证人类 | 完整阅读 ≥3 篇 |
| Level 3 | 活跃关系 | 曾直接回复邮件 |
| Level 4 | 核心圈 | 6 个月以上 + 多次互动 |

## 二、人类订阅者旅程

### 1. 发现与订阅

订阅者通过以下方式找到创作者的 Space：

- **落地页**：`/{creator-slug}` — 展示创作者简介、统计数据、最新内容列表
- **外部网站嵌入表单**：创作者可将订阅表单嵌入其博客、官网等外部网站（每个网站独立 embed key，订阅来源自动标记）

订阅表单填写：
- 邮箱（必填）
- 姓名（可选）

提交后系统自动发送欢迎邮件（如创作者已启用）。系统同时记录订阅来源（来自落地页还是哪个外部网站），方便创作者分析渠道效果。

### 2. 接收内容

**邮件渠道**：
- 每期内容以格式精良的 HTML 邮件送达（仅当该内容开放邮件渠道时）
- 邮件包含 AI 摘要区块，忙碌时快速获取精华
- 正文全文，无需跳转外链

**网页阅读**：
- 点击落地页的内容列表，直接进入文章页（仅展示开放落地页渠道的内容）
- 文章页包含：AI 摘要（蓝色区块）、核心论点（编号列表）、完整正文、作者注释
- 文章底部提供订阅入口

### 3. 管理订阅

邮件底部包含「取消订阅」链接，点击即退订，不再接收后续邮件。

如误操作退订，可回到落地页重新填写邮箱完成再订阅。

## 三、AI Agent 订阅者旅程

Agent 以 API Key 代替邮箱订阅，通过结构化 API 获取内容（非 HTML 邮件格式）。

### 1. 注册 Agent 订阅

```http
POST /api/v1/agent/subscribe
Content-Type: application/json

{
  "agent_id": "my-personal-assistant",
  "agent_type": "personal_assistant",
  "preferred_format": "structured_json",
  "query_frequency": "on_demand"
}
```

返回 API Key，格式为 `lttr_xxxxxxxxxxxx`。

### 2. 获取内容

**内容列表**（含摘要和核心论点）：
```http
GET /api/v1/feed
Authorization: Bearer lttr_xxxxxxxxxxxx
```

注：仅返回已开放「落地页」渠道的内容（等同公开可见内容）。

**单篇全文**（Markdown + 结构化字段）：
```http
GET /api/v1/issues/{id}
Authorization: Bearer lttr_xxxxxxxxxxxx
```

**语义搜索**：
```http
GET /api/v1/search?q=关键词
Authorization: Bearer lttr_xxxxxxxxxxxx
```

### 3. 内容格式

API 返回标准化 JSON，包含：

```json
{
  "id": "uuid",
  "title": "标题",
  "summary": "AI 摘要，专为机器阅读优化",
  "key_points": ["核心论点 1", "核心论点 2", "核心论点 3"],
  "topics": ["AI", "产品"],
  "full_content_markdown": "...",
  "full_content_html": "...",
  "author_note": "创作者手写注释",
  "reading_time_minutes": 8,
  "content_penetration_score": 78,
  "published_at": "2026-02-24T10:00:00Z"
}
```

## 四、MCP 订阅者旅程

MCP 订阅者将 Space 作为工具接入 Claude Desktop、Cursor 等 AI 客户端，在与 AI 对话时实时检索内容。

### 1. 接入配置

在创作者落地页的「MCP 接入」区块，复制配置代码，粘贴至 AI 客户端配置文件：

**Claude Desktop 配置示例**（`claude_desktop_config.json`）：
```json
{
  "mcpServers": {
    "creator-slug": {
      "url": "https://your-domain.com/api/mcp",
      "headers": {
        "x-creator-slug": "creator-slug"
      }
    }
  }
}
```

配置保存后重启 AI 客户端，即完成接入。

### 2. 使用方式

接入后，在 AI 对话中直接提问：

- **「最近有什么新内容？」** → 调用 `list_issues`，返回最新内容列表和摘要
- **「帮我找关于 AI 产品的文章」** → 调用 `search_content`，语义搜索相关内容
- **「获取这篇文章的全文」** → 调用 `get_issue`，返回完整正文（Markdown 格式）
- **「这个 Space 是关于什么的？」** → 调用 `get_author_info`，返回创作者简介

### 3. 可用工具

| 工具名 | 功能 |
|--------|------|
| `list_issues` | 获取最新内容列表（含摘要、核心论点） |
| `get_issue` | 获取特定内容的完整正文（Markdown） |
| `search_content` | 语义搜索历史内容 |
| `get_author_info` | 获取创作者信息和 Space 简介 |

## 五、各角色旅程对照

```
创作者
  │ 登录 → 配置设置（落地页 + 外部网站 + 邮件服务）
  │   ↓
  │ 创建内容 → 选择渠道（落地页 / 邮件 / 各网站）→ 发布
  │   ↓                                              ↓
  │ 在内容列表按渠道筛选内容               触发邮件发送
  └────────────────────────────→ 查看双轨分析仪表盘
                                  ├── 各渠道订阅增长
                                  └── Agent / MCP 查询趋势

人类订阅者
  │ 发现落地页 或 外部网站嵌入表单
  │   ↓
  │ 填写邮箱 → 收到欢迎邮件（来源自动标记）
  └── 后续：邮件直达 / 落地页浏览文章 / 回复邮件互动

AI Agent 订阅者
  └── 注册 API Key → 拉取公开渠道内容 → 集成到工作流或 AI 应用

MCP 订阅者
  └── 配置 Claude Desktop → 在对话中实时检索内容 → 获取创作者观点和分析
```

## 六、关键 URL 结构

| 路径 | 内容 | 访问权限 |
|------|------|---------|
| `/{slug}` | 创作者落地页 | 公开 |
| `/dashboard` | 创作者管理后台 | 需登录 |
| `/dashboard/content` | 内容列表（支持状态+渠道双重筛选） | 需登录 |
| `/dashboard/content/new` | 新建内容 | 需登录 |
| `/dashboard/content/{id}` | 编辑内容（含渠道选择器） | 需登录 |
| `/dashboard/subscribers` | 订阅者管理 | 需登录 |
| `/dashboard/analytics` | 分析仪表盘 | 需登录 |
| `/dashboard/settings` | 设置页面（含各渠道订阅者数） | 需登录 |
| `/embed/{key}` | 外部网站订阅表单（iframe 嵌入） | 公开 |
| `/api/v1/feed` | JSON Feed（落地页渠道内容） | 公开 |
| `/api/v1/issues/{id}` | 单篇内容 API | 公开 |
| `/api/v1/search` | 内容搜索 API | 公开 |
| `/api/mcp` | MCP Server 入口 | 公开 |
| `/unsubscribe` | 退订页面 | 带 token 的链接 |
